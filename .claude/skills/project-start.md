# Project Start — 开发会话启动

每次新开 Claude Code 对话继续项目开发时使用。

## 执行步骤

### 1. 确认位置

```bash
pwd
```

如果不在项目根目录，提醒用户 `cd` 到正确路径。

### 2. 读项目上下文

按顺序读取以下文件（如存在）：

1. `CLAUDE.md` — 项目入口文档（技术栈、目录结构、开发规范）
2. `progress.json` — 任务进度追踪
3. `DEV_WORKFLOW.md`（如有）— 开发工作流说明

### 3. 读上次进度

```bash
# 看最近提交
git log --oneline -5

# 看未提交的改动
git status --short
```

如果存在未提交改动，提醒用户决定：提交 or 暂存（git stash）。

### 4. 分析进度

读取 `progress.json`：

- 统计：已完成数 / 总数
- 找到第一个 `passes: false` 的条目
- 确认该任务的前置依赖已完成（dependencies 字段）

### 5. 验证当前状态

根据当前任务所属的 phase/阶段，快速验证前置产出是否就绪：

```bash
# 通用检查：看最近修改的文件
find . -newer progress.json -type f -not -path './.git/*' -not -path './.claude/*' 2>/dev/null

# 看关键目录是否有内容
ls -lt <项目关键输出目录> 2>/dev/null | head -5
```

> 这一步的具体检查项应该在 CLAUDE.md 的「验证」部分定义。
> 如果没有定义，跳过此步。

### 6. 输出状态报告

```
项目状态
━━━━━━━━━━━━━━━━━━━━━━━━
上次完成: [git log 最近一条相关信息]
整体进度: X/N 完成 (Phase/K阶段 名称)
下一个任务: [ID] - [任务名称]
  描述: [任务描述]
  前置状态: ✅ 已满足 / ⚠️ 未满足

建议本次实现 [任务ID]，计划：
  [简要描述实现方案]

确认后我开始。完成后输入 /project-done 提交。
```

### 7. 等待用户确认

**不要自动开始写代码。** 先报告状态，等用户说「开始」或给出具体指令。

---

## 重要原则

- 只读不改：启动阶段不修改任何代码
- 快速恢复：整个启动流程控制在 3-5 步内
- 人审优先：状态报告后必须等人确认
- 如果 progress.json 不存在或为空，提示用户先做规划阶段
