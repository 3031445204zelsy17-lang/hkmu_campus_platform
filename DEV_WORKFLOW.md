# Claude Code 长任务开发工作流

> 适用于：任何需要多个 session 才能完成的项目（3 个任务以上）
> 灵感来源：ole-scraper（15 功能）、STAT2610（30 任务）的实际开发经验

---

## 核心理念

**一次对话 = 一个原子任务。** 每次只做一件事，做完提交，更新进度。下次对话从 JSON 恢复状态。

为什么这样做：
- Claude Code 有上下文窗口限制，长任务会越来越慢
- 每次只做一个任务，上下文始终干净
- JSON 进度文件是跨 session 的"记忆"
- git commit 是每次工作的安全检查点

---

## 三个阶段

```
阶段1: 规划（Plan）
  └─ 理解需求 → 拆分为任务 → 写 progress.json → 人审

阶段2: 执行（Act）← 重复 N 次，每次一个任务
  └─ /project-start → 写代码 → 测试 → /project-done

阶段3: 验收（Review）
  └─ 全量测试 → 人审 diff → 清理 → 更新项目记忆
```

---

## 阶段 1: 规划（首次对话）

### 你做什么

在 Claude Code 中描述你的项目需求和目标。Claude 会帮你：

1. 分析需求，拆分为有序任务
2. 生成 `progress.json`（所有任务标为 passes: false）
3. 生成或更新 `CLAUDE.md`（项目入口文档）
4. 初始化 git，首次提交

### 你审查什么

- `progress.json` 的任务是否覆盖你想做的所有功能
- 任务顺序是否合理（前置依赖是否排前面）
- 确认后说「确认，进入执行」

---

## 阶段 2: 执行（每个任务一次对话）

### 开场

输入 `/project-start`，它会自动：
1. 确认你在正确的项目目录
2. 读取 git log 和 progress.json
3. 报告上次进度和下一个待完成任务
4. 等你确认后开始

### Claude 做什么

```
┌─ 恢复上下文
│  git log → 上次做了什么
│  progress.json → 该做什么
│
├─ 原子操作
│  只改这一个任务相关的文件
│  不动其他已完成的代码
│
├─ 验证
│  运行测试或手动检查
│  确认功能正常
│
├─ HITL 检查点
│  展示改动，等你说「继续」或「改一下 xxx」
│
└─ 提交
   git add 具体文件
   git commit -m "feat: T0X - 描述"
   更新 progress.json
   git commit -m "progress: T0X done"
```

### 收尾

输入 `/project-done`，它会自动：
1. 验证任务完成情况
2. 查看改动范围
3. 更新 progress.json
4. 输出完成报告和下一个任务

---

## 阶段 3: 验收（全部完成后）

输入 `/project-review`，或手动：
1. 读 progress.json，确认所有 passes: true
2. 全量测试
3. 发现问题 → 回到阶段 2 修复
4. 全部通过 → 最终提交

---

## 上下文管理

### 对话变长时

在对话中输入：
```
总结当前进度，清理对话历史。
```

### 跨对话时

用 `/project-start` 自动恢复。它读 git log + progress.json 重建上下文。

### 任务颗粒度建议

| 项目规模 | 建议颗粒度 | 示例 |
|---------|-----------|------|
| 小（3-8 任务） | 按功能模块 | 认证系统、帖子系统 |
| 中（8-20 任务） | 按功能子模块 | 用户注册、用户登录、JWT 中间件 |
| 大（20+ 任务） | 按可验证的最小单元 | 创建用户表、注册 API、注册页面 |

原则：每个任务的结果都应该是**可独立验证的**（能跑测试、能看到效果）。

## 文件结构

```
你的项目/
├── .claude/
│   ├── skills/              ← 开发技能
│   │   ├── project-start.md   每次 session 启动
│   │   ├── project-done.md    每次任务完成
│   │   └── project-review.md  最终验收
│   └── settings.local.json  ← 项目设置
├── progress.json            ← 任务进度追踪（核心）
├── CLAUDE.md                ← 项目入口文档
└── ...                      ← 你的代码
```
