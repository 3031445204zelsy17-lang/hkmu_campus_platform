### P2｜体验打磨 72→90（决定"成熟感"，用户感知层）

#### FR5 小程序静默失败补 toast（私信/聊天） `P2 quick win` `SYF`
- [ ] **code-done** — 精准修主加载失败：messages.js `loadConversations`(!silent 时 toast) + chat.js `loadHistory`(reset 时 toast)；best-effort 的 fetchUnread/markRead 保持静默（合理）。node --check 全过
- **问题**：私信/聊天多处网络错误只静默 catch，用户完全不知道出错了。
- **证据**：`messages.js:125` loadConversations 失败静默；`chat.js:97,112,161,192` 多处 `catch(()=>{})` 不提示。
- **做法**：catch 里补 `wx.showToast`（复用 `loadFail`/`actionFail` i18n）。
- **验收**：私信/聊天请求失败 → toast 提示（非静默）；断网场景可见提示。
- **难度**：小，高收益。**建议优先做。**

#### FR6 小程序图片加载占位符（对齐 web onerror） `P2 quick win` `SYF`
- [ ] **code-done（核心 feed）** — community + home：image 加 `binderror`，失败置空 → 走已有 `authorInitial` 字母兜底 / 隐藏配图（复用现有兜底，无需默认头像）。范围外后续：post-detail 配图、messages/chat/new-chat 头像（需补 initial 兜底）。node --check + wxml 自闭合确认
- **问题**：小程序端头像/帖子配图加载失败显示破损图；网页端已有 `img.onerror` fallback，小程序缺。
- **证据**：web `community.js:134` img.onerror 头像 fallback 已有；小程序 `<image>` 无 `binderror`。
- **做法**：各 `<image>` 加 `binderror` → fallback 占位图/隐藏。对齐网页端逻辑。
- **验收**：头像/配图加载失败 → 占位图或隐藏（非破损图标）。
- **难度**：中，高收益。**建议优先做。**
