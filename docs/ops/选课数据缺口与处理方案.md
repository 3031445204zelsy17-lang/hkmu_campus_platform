## 1. 问题

| 代码 | 专业 | programmes.py 状态 |
|------|------|-------------------|
| `BSCHDSAIJ` | 数据科学及人工智能 | ✅ 完整(43 门课、6 类别、min_credits 全填、有 template) |
| `BSCHCSJ` | 计算机科学 | 🔶 `coming_soon=True`,全空 |
| `BSCHCCSJ` | 网络安全及计算机保安 | 🔶 `coming_soon=True`,全空 |

而资料库 `docs/ops/选课选课/skill.md`(+ 6 个学院 docx)**覆盖全校**,落差极大:

| | 资料库 | programmes.py 接了 |
|--|--------|--------------------|
| 学院 | 5 | 1(科技) |
| 专业(带代码) | 106 | 3 |
| 课程条目 | 4768(含跨专业重复) | 43(只 DSAI) |

资料库按学院分布:商管 37 / 科技 28 / 文艺社科 20 / 护理健康 12 / 教育与语言 9。

## 2. 真实性核验(三路交叉,2026-07-03)

**结论:作为"课程目录"真实可信;作为"选课规划/毕业计算"数据不全。**

| 核验途径 | 结果 |
|---------|------|
| HKMU 官网 | CS 专业专页存在 `hkmu.edu.hk/st/computing/programmes/full-time/3-credit-unit/bschcsjs/`;3CRU PDF `3CRU_FTU_ST_BSCHCSJ.pdf` 存在 |
| 具体课程 | `COMP S208F Introduction to Computer Programming` 官方确认 5 学分、Middle level,与 skill.md 吻合;后续 `COMP S209F`(5学分)亦吻合 |
| 本地 docx | 科技学院 docx 为 29 表格,来源标注 "official publicly accessible HKMU Programme Requirements PDFs";skill.md 是其忠实结构化产物 |

**真实性分层(关键):**

| 数据 | 资料里有 | 选课规划需要 |
|------|---------|-------------|
| 课程代码/名称/学分 | ✅ 真 | ✅ |
| 课程归属(专业/官方分组) | ✅ 真 | ✅ |
| 学期(autumn/spring/学年) | ❌ 全 "Not specified in public PDF" | ✅ 核心 |
| 先修课 prerequisites | ❌ 只在 MyHKMU | ✅ 核心 |
| 分类毕业 min_credits | ⚠️ 仅总学分 120,分项要人工推 | ✅ 核心 |

skill.md 与 docx 头部均明确声明 limitation:"class schedules and pre-requisites are provided through MyHKMU > Programme > Classes & Enrolment"。

**洞察**:缺的三项恰是选课规划最依赖的。DSAI 现在能算毕业,是当初人工对照官方 Study Plan 补的(memory `hkmu-dsai-curriculum`)。直接灌 skill.md 做 CS/网安 → 课程清单真,但学期是猜的、先修空、分类学分凭感觉 → 学生按它规划却毕不了业。

## 3. 待解决:三条处理路径

### ❌ 路径 1:硬补 CS/网安 假规划
把 skill.md 数据灌进 programmes.py + seed_courses,编学期/留空先修/估 min_credits。
- **不推荐**。算错毕业进度比 `coming_soon` 占位更伤信任,经不起老师/学生追问。

### ✅ 路径 2:课程目录层(推荐)
- **数据隔离**:新建只读 `course_catalogue` 表(字段:school / programme_code / programme_name / official_group / course_code / course_name / credits),**不灌进**现有 `courses` 规划表、不扩 `PROGRAMMES` 的 min_credits。
- **一个入口两种深度**:复用 planner 专业下拉。选 DSAI → 现有完整规划;选其他 105 个 → "课程目录"视图(按官方分组列课程 + 一行"该专业选课规划开发中,以下为官方课程目录")。
- **诚实标注时效**:固化"数据采集自 HKMU 官方 PDF(2026-06),仅供参考,以 MyHKMU 为准"。
- **工作量**:解析脚本 + 表 + 查询端点(中);小程序/网页 planner 加目录视图分支(中)。一次会话可出 MVP。
- **三个必处理坑**:
  1. 假专业裁剪:`BFAHIDDAJ`(341门)、`BSCHSTEMJ`(296门)、`STEAM`(170门)是"综合大类"堆叠,解析时按课程数异常高识别并降级/不收。
  2. 两套代码:5-credit(`COMP S208F`)与 3CRU(`COMP 1080SEF`)并存,目录层可都收(只展示),但须跨专业去重。
  3. 中英文名:skill.md 全英文,先英文或按 programme_code 做中英映射。

### 退路:只改文案(最小代价)
把 CS/网安 `coming_soon` 文案改成诚实声明:*"选课规划功能当前仅 DSAI 专业可用,更多专业开发中"*。零开发、零风险、也诚实。内测能跑通不丢人。

## 4. 推荐

- **要做点的** → 走路径 2(课程目录层):真数据、低风险、能讲"接入全校 5 学院 106 专业官方课程资料"的故事,同时不在毕业计算上撒谎。
- **想聚焦社交内测** → 走退路(改文案)。
- **唯一不建议**:为"看着专业多"硬补带假学期/假先修的规划。
